# Copy-Only Generator

Create the shortest possible structured spec.

Rules:
- Restate the task as the goal.
- Include only the inputs and output that are obvious from the task text.
- Put the visible example/assertion in acceptance criteria if one exists.
- Leave edge cases, invariants, and extra constraints empty unless the task
  explicitly states them.
- Do not infer hidden cases or broader programming-challenge semantics.

When revising, make only the smallest change needed for the issue.
